# Инструкция: Настройка и Деплой "Капли"

Это PWA работает полностью в браузере и общается напрямую с базой данных Firebase. Чтобы оно заработало, вам нужно создать свой проект Firebase.

## 1. Регистрация и настройка Firebase

1. Перейдите в [Firebase Console](https://console.firebase.google.com/).
2. Нажмите **Add project** (Добавить проект). Задайте имя (например, `kapli-tracker`). Можно отключить Google Analytics для этого проекта.
3. По завершении создания нажмите на **Web**-иконку (`</>`) на главном экране проекта, чтобы добавить веб-приложение.
   - Дайте ему название (просто `Kapli App`).
   - Можно сразу поставить галочку "Also set up Firebase Hosting", но мы рассмотрим и более простые способы деплоя.
   - Нажмите **Register app**.
4. Firebase выдаст вам объект конфигурации `firebaseConfig`. Он выглядит примерно так:
   ```javascript
   const firebaseConfig = {
     apiKey: "AIzaSyB...",
     authDomain: "kapli-tracker.firebaseapp.com",
     projectId: "kapli-tracker",
     storageBucket: "kapli-tracker.appspot.com",
     messagingSenderId: "123...",
     appId: "1:123...:web:abc..."
   };
   ```
5. Скопируйте содержимое объекта и вставьте его в файл `app.js` (найдите закомментированный кусок `// TODO: Замените этот объект на свой Firebase Config` и замените заглушку на ваш реальный конфиг).

## 2. Создание Firestore Database

1. В боковом меню консоли Firebase выберите **Firestore Database**.
2. Нажмите **Create database** (Создать базу данных).
3. Выберите локацию (ближайшую к вам, например, `europe-west3`).
4. На этапе выбора правил безопасности выберите **Start in test mode** (Тестовый режим).
   - *Важно:* В тестовом режиме база открыта для всех на 30 дней. Если вы планируете пользоваться приложением долго, после истечения срока измените правила во вкладке "Rules" на:
     ```
     rules_version = '2';
     service cloud.firestore {
       match /databases/{database}/documents {
         match /{document=**} {
           allow read, write: if true; 
         }
       }
     }
     ```
     *(Так как приложение без авторизации, это правило дает доступ всем, у кого есть конфиг. Конфиг публичный в JS-файле, поэтому кто угодно сможет писать в базу. Для личного PWA это не так страшно, но имейте в виду).*
5. Все готово! Коллекция `drops_history` создастся автоматически при первом добавлении записи.

## 3. Деплой приложения

Поскольку приложение состоит только из статических файлов (`html`, `css`, `js`), вы можете задеплоить его бесплатно и быстро:

### Способ 1: GitHub Pages
1. Инициализируйте Git в папке `Kapli`, закоммитьте файлы и запушьте в новый репозиторий на GitHub.
2. В репозитории зайдите в **Settings** -> **Pages**.
3. В разделе Build and deployment выберите "Deploy from a branch", выберите `main` или `master`, и нажмите **Save**.
4. Через пару минут приложение будет доступно по адресу `https://<ваш_ник>.github.io/<название_репо>/`.
   *(Важно: если приложение лежит не в корне, возможно, потребуется поправить пути в `manifest.json` и `sw.js` с `./` на относительные пути репозитория).*

### Способ 2: Vercel / Netlify
1. Зарегистрируйтесь на [Vercel](https://vercel.com/) или [Netlify](https://netlify.com/) (войдите через свой GitHub).
2. Нажмите **Add New...** -> **Project**.
3. Импортируйте репозиторий с проектом из GitHub или просто перетащите папку с файлами (Drop folder) прямо в интерфейсе Netlify.
4. Никаких настроек сборки не требуется — просто оставьте поля `Build Command` пустыми.
5. Вы получите быструю, защищенную https-ссылку.

## 4. Установка на iPhone / iPad

1. Откройте полученную ссылку (Vercel, GitHub Pages или Firebase Hosting) в браузере **Safari** на iPhone или iPad.
2. Нажмите на иконку "Поделиться" (квадрат со стрелочкой вверх).
3. В меню выберите **На экран «Домой»** (Add to Home Screen).
4. Назовите приложение и нажмите "Добавить".
5. Теперь на рабочем столе появилась иконка приложения. Оно будет открываться на весь экран без интерфейса браузера, как нативное. Синхронизация данных между устройствами (при подключенном интернете) будет происходить мгновенно.
